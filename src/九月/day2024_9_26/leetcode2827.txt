
2827中取模的理解：
这是一个很好的问题！要理解 (val * 10 + d) % k 这个操作的最终结果如何保证与原数 % k 的结果相同，我们可以从数的性质和取模的性质出发进行分析。

让我们假设 val 是当前已经构建的数字的一部分，并且当前的数字可以表示为 X，也就是在这个状态下，数字的完整形式为 X。接下来，我们要添加的数字是 d。可以想象此时我们有一个数字 X，接下来我们要构建 Xd，即在 X 后面追加 d。

1. 数字构建
假设 X 可以表示为一个数字，其在十进制下可以表示为：

如果 X 是一个 n 位数字，那么 X 的值为 abc...，即 X 的值实际上是 a * 10^(n-1) + b * 10^(n-2) + ... + c * 10^0。
当我们在 X 后面追加 d，我们得到的数字 Xd 表示为：
[
Xd = X \times 10 + d
]

2. 取模性质
取模运算有一个重要的性质，就是对于任意整数 a 和 b，有：
[
(a + b) % k = ((a % k) + (b % k)) % k
]

根据上述性质，我们可以将我们的操作进行分解：

计算 X % k：

我们设 X % k = val，那么 val 是当前构建的数字的余数。
计算 Xd % k：

我们需要计算 (X * 10 + d) % k。
根据取模性质，我们可以分解为： [ (X * 10 + d) % k = ((X * 10) % k + d) % k ]
进一步分析 X * 10 % k：

由于 X % k = val，我们有： [ (X * 10) % k = (val * 10) % k ]
因此，得到： [ (X * 10 + d) % k = ((val * 10) % k + d) % k ]
最终结果：

将上面的表达式结合起来，我们得到了： [ (X * 10 + d) % k = ((val * 10) + d) % k ]
3. 结论
因此，通过以上的步骤，我们可以看到 (val * 10 + d) % k 实际上在数学上是等同于 (X * 10 + d) % k 的。也就是说，当我们在构建数字时，保持 val 的值乘以 10 并加上新数字 d，然后取模 ( k ) 的结果，实际上是正确地反映了完整数字 Xd 与 ( k ) 的余数。

d不取余主要是默认有k>d,故要根据实际来进行判断到底应不应该来在d后面添加%k的符号，不过默认添加也无所谓
